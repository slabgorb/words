# Retrospective Definition of Done

## Epic Review Complete
- [ ] Epic number confirmed with user
- [ ] All story files analyzed for lessons and patterns
- [ ] Previous retrospective reviewed (if exists)
- [ ] Successes and challenges discussed with team
- [ ] Key insights synthesized

## Next Epic Preparation (if applicable)
- [ ] Next epic previewed and dependencies identified
- [ ] Preparation tasks defined with owners
- [ ] Critical path items documented
- [ ] Knowledge gaps identified

## Action Items Captured
- [ ] Process improvement actions defined with owners
- [ ] Technical debt items prioritized
- [ ] Documentation needs identified
- [ ] Team agreements documented

## Readiness Verified
- [ ] Testing and quality status confirmed
- [ ] Deployment status clarified
- [ ] Stakeholder acceptance status checked
- [ ] Unresolved blockers documented

## Documentation Complete
- [ ] Retrospective summary document saved
- [ ] Sprint status updated (retrospective marked done)
- [ ] Action items ready for standup review
