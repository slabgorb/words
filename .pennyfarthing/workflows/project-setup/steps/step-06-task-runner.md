# Step 6: Create Task Runner (justfile/Makefile)

<purpose>
Create a justfile or Makefile in the orchestrator that proxies commands to subrepos, providing a single entry point for common operations.
</purpose>

<instructions>

</output>

## TASK RUNNER DETECTION

Check for existing task runners:

```bash
ls -la justfile Justfile Makefile makefile package.json 2>/dev/null
```

### Prefer justfile over Makefile

**Why justfile:**
- Modern syntax, easier to read
- Better error messages
- Cross-platform (Windows support)
- Native variables and interpolation
- Recipes are isolated by default

## JUSTFILE STRUCTURE FOR ORCHESTRATORS

```just
# justfile for {project_name}
# Generated by Pennyfarthing project-setup workflow

# Default recipe - show help
default:
    @just --list

# ============================================
# ALL REPOS
# ============================================

# Run tests in all repos
test-all:
    @echo "Testing all repos..."
    just test-api
    just test-ui

# Build all repos
build-all:
    @echo "Building all repos..."
    just build-api
    just build-ui

# Install dependencies in all repos
install-all:
    @echo "Installing dependencies..."
    cd {api_path} && npm install
    cd {ui_path} && npm install

# ============================================
# API ({api_path})
# ============================================

# Run API tests
test-api:
    cd {api_path} && {api_test_command}

# Build API
build-api:
    cd {api_path} && {api_build_command}

# Start API development server
dev-api:
    cd {api_path} && {api_dev_command}

# Lint API
lint-api:
    cd {api_path} && {api_lint_command}

# ============================================
# UI ({ui_path})
# ============================================

# Run UI tests
test-ui:
    cd {ui_path} && {ui_test_command}

# Build UI
build-ui:
    cd {ui_path} && {ui_build_command}

# Start UI development server
dev-ui:
    cd {ui_path} && {ui_dev_command}

# Lint UI
lint-ui:
    cd {ui_path} && {ui_lint_command}

# ============================================
# ORCHESTRATOR
# ============================================

# Start both API and UI (background API, foreground UI)
dev:
    just dev-api &
    just dev-ui

# Run linting in all repos
lint-all:
    just lint-api
    just lint-ui

# Clone missing subrepos
clone-repos:
    @echo "Cloning subrepos..."
    test -d {api_path} || git clone {api_git_url} {api_path}
    test -d {ui_path} || git clone {ui_git_url} {ui_path}

# Check repo status across all repos
status:
    @echo "=== Orchestrator ===" && git status --short
    @echo "=== API ===" && cd {api_path} && git status --short
    @echo "=== UI ===" && cd {ui_path} && git status --short
```

## GENERATION FROM REPOS.YAML

For each repo in repos.yaml, generate recipes:

```yaml
# repos.yaml input:
repos:
  api:
    path: conductor-api
    test_command: npm test
    build_command: npm run build
    dev_command: npm run dev
    lint_command: npm run lint
```

Generates:
```just
test-api:
    cd conductor-api && npm test

build-api:
    cd conductor-api && npm run build
```

## COMMON ORCHESTRATOR RECIPES

### Framework Recipes (from `.pennyfarthing/justfile.pf`)

The root justfile imports `.pennyfarthing/justfile.pf` which provides these recipes automatically:

```just
# ============================================
# PENNYFARTHING FRAMEWORK (auto-imported)
# ============================================

# Show current sprint status
sprint:
    pf sprint status

# List available stories
backlog:
    pf sprint backlog

# Start Frame server (API + WebSocket + OTEL ingestion)
frame

# Open Frame dashboard in browser (starts Frame if needed)
frame

# Launch TUI panel viewer (starts Frame if needed)
tui

# Launch Claude with OTEL pre-configured for Frame
claude

# Launch tmux dev layout (Claude + TUI in split panes)
start

# Run any pf CLI command
pf <command>
```

These recipes delegate to `pf launch` commands and handle server lifecycle (start, stop, port detection) automatically.

### CI/CD Recipes

```just
# ============================================
# CI/CD
# ============================================

# Run full CI check
ci: lint-all test-all build-all
    @echo "CI passed!"

# Create release
release version:
    @echo "Creating release {{version}}..."
    # Add release logic here
```

## MAKEFILE ALTERNATIVE

If user prefers Makefile:

```makefile
# Makefile for {project_name}
# Generated by Pennyfarthing project-setup workflow

.PHONY: test-all build-all install-all test-api build-api dev-api

# Default target
help:
	@echo "Available targets:"
	@grep -E '^[a-zA-Z_-]+:.*?## .*$$' $(MAKEFILE_LIST) | sort | awk 'BEGIN {FS = ":.*?## "}; {printf "  %-20s %s\n", $$1, $$2}'

test-all: test-api test-ui ## Run tests in all repos

test-api: ## Run API tests
	cd {api_path} && {api_test_command}

test-ui: ## Run UI tests
	cd {ui_path} && {ui_test_command}
```
