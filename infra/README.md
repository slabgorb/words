# infra

Hosting bits that live outside the project tree on a running host but should
survive a fresh clone.

## Files

- `launchd/com.slabgorb.words-server.plist` — runs `node src/server/server.js`
  under launchd, autostart on login, restart on crash. Uses `__HOME__` /
  `__PROJECT_DIR__` placeholders that `just install` substitutes.
- `launchd/com.slabgorb.words-tunnel.plist` — runs `cloudflared tunnel run words`
  under launchd. Same placeholder pattern.
- `cloudflared/words-config.yml.example` — template for the tunnel config that
  needs to live at `~/.cloudflared/words-config.yml`. Per-machine because the
  tunnel UUID is generated by `cloudflared tunnel create`.

## Bootstrap on a fresh host

```bash
# 1. Install runtime deps
brew install node cloudflared just

# 2. Authenticate cloudflared (browser pops, drops cert.pem in ~/.cloudflared)
cloudflared tunnel login

# 3. Create the tunnel + DNS route
cloudflared tunnel create words
cloudflared tunnel route dns --overwrite-dns words words.slabgorb.com

# 4. Write ~/.cloudflared/words-config.yml from the example, plugging in the UUID
cp infra/cloudflared/words-config.yml.example ~/.cloudflared/words-config.yml
$EDITOR ~/.cloudflared/words-config.yml

# 5. App setup
npm install
npm run fetch-dict

# 6. Materialize the LaunchAgent plists and start both services
just install
just up
just status
```

`just install` substitutes `__HOME__` / `__PROJECT_DIR__` and writes the result
to `~/Library/LaunchAgents/`. Re-running it is safe; it overwrites the
installed plist with the current template.
